## 2. 自动化测试

测试文件：`tests/test_recommendation.py`

| 用例 | 目标 | 预期 |
| --- | --- | --- |
| `test_recommend_im_returns_at_least_three_candidates` | 即时通讯需求推荐 | 返回 3 个候选，核心推荐属于事件驱动/微服务/CQRS |
| `test_styles_include_required_knowledge_base_size` | 知识库规模 | 架构风格数量不少于 10 |
| `test_knowledge_graph_has_nodes_and_edges` | 知识图谱 | 返回节点和边 |
| `test_service_generates_matrix_and_trace` | 决策输出 | 生成对比矩阵和 Agent trace |

运行命令：

```powershell
pytest
```

## 4. 手工验收步骤

1. 启动系统：`uvicorn app.main:app --reload --host 127.0.0.1 --port 8000`
2. 打开 `http://127.0.0.1:8000`
3. 使用默认即时通讯需求点击“生成推荐”
4. 验证页面展示：
   - 最终推荐
   - 需求特征
   - 候选架构对比矩阵
   - 评估报告
   - Agent 协作追踪
   - 架构拓扑图
5. 打开 `http://127.0.0.1:8000/docs` 验证 Web API

## 5. 风险与改进

- 当前知识图谱为轻量 JSON/内存模型，生产级可替换为 Neo4j。
- 中文特征抽取采用规则启发式，后续可接入分词模型或 Embedding 检索。
- LLM 调用依赖外部 Key，系统已提供无 Key 降级策略。
